Mapping Students’ Commutes to School

SDI Project 2025 Spatial Data Infrastructure

Overview

This project visualizes how students in Austria commute to school, focusing on accessibility, travel time, and district-level disparities. Using openly available data and SDI principles, we built a complete workflow: data collection, preprocessing, database integration, web service publication, and dashboard visualization.

The goal was to support education planners by highlighting where long commutes occur, which school types are underserved, and how commuting patterns differ across districts.

Background

School types in Austria are unevenly distributed, forcing many students to travel long distances. Long commutes reduce free time, increase stress, and contribute to environmental impact. Despite this, visualizations of school commutes are rare, especially for specialized schools.

By mapping commute distances, times, and origin–destination flows, the project provides insights into structural shortcomings and supports more equitable educational planning.

Purpose

The dashboard presents:

  • Choropleth maps of commuter counts per district
  • Origin–destination lines between districts
  • Filters for school type, district, and year
  • Pop‑ups with district-level statistics

All data is stored and served through standardized, open geospatial services.

Project Workflow

The project followed a complete SDI workflow, starting from data acquisition and preprocessing, moving through database integration and service publication, and ending with a fully interactive dashboard.

  1. Data Collection: Gathering vector and statistical datasets from Statistik Austria and data.gv.at.
  2. Preprocessing: Cleaning, harmonizing, and transforming data using Python, Pandas, NumPy, and GDAL.
  3. Database Integration: Uploading spatial and tabular data into a PostGIS-enabled PostgreSQL database.
  4. Web Service Publication: Exposing vector layers as WFS and statistical tables as REST services.
  5. Dashboard Implementation: Building an ArcGIS Insights dashboard with real-time filtering and OD visualization.

Database Structure

The database was implemented using PostgreSQL/PostGIS, designed around a single district identifier (bezirk_id) that links all spatial and statistical datasets. This ensured seamless joins between layers and efficient querying from the dashboard.

Spatial Tables

  • districts — Polygon boundaries of Austria’s political districts.
  • district_centroids — Precomputed centroid points for OD routing.
  • school_locations — Point dataset of Austrian schools, categorized by type.

Statistical Tables

  • commute_time — Average commute time per district and school type.
  • commute_distance — Average commute distance per district and school type.
  • commute_characteristics — Detailed breakdown of commute modes and durations.
  • school_attendance — Number of students per district and school type.
  • od_matrix — Origin–destination flows between districts.

Migration & Transformation

Data was initially stored in Supabase but migrated to an ArcGIS-enabled PostGIS instance using GDAL: 

ogr2ogr -f "PostgreSQL" PG:"host=... dbname=... user=..." \
    input.geojson -nln districts -nlt MULTIPOLYGON

Indexes were added for performance:

CREATE INDEX idx_districts_geom ON districts USING GIST (geom);
CREATE INDEX idx_od_origin ON od_matrix (bezirk_id);
CREATE INDEX idx_od_dest ON od_matrix (bezirk_id_dest);

Web Services

All spatial layers were published as OGC-compliant Web Feature Services (WFS) through the university’s GeoServer instance. Statistical tables were exposed as REST services.

  • districts_wfs — polygon boundaries
  • district_centroids_wfs — centroid points
  • school_locations_wfs — school locations
  • commute tables — REST-accessible statistical data

Dashboard Workflow

The dashboard was built in ArcGIS Insights, combining spatial and non-spatial data with interactive filtering and real-time aggregation.

1. Choropleth Mapping

  1. Load districts_wfs.
  2. Join with statistical tables via bezirk_id.
  3. Apply filters for school type and year.
  4. Style polygons using quantile or natural breaks.
  5. Enable pop-ups with district summaries.

2. Origin–Destination Flow Mapping

  1. Load district_centroids_wfs.
  2. Join OD matrix to centroid coordinates.
  3. Generate dynamic line geometries.
  4. Filter by flow intensity.
  5. Enable pop-ups showing commuter counts.

3. Analytical Panels

  • Commute time and distance summaries
  • School attendance statistics
  • Commute characteristics by education level
  • Contextual explanations for non-technical users
